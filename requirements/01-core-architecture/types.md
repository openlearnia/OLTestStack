# Shared Types

> **Module:** `01-core-architecture`  
> **Feature:** Shared data models and type definitions  
> **Depends on:** [system-architecture.md](./system-architecture.md)  
> **Depended on by:** All modules (02â€“13)

## Overview / Purpose

Define the canonical TypeScript interfaces and shared data shapes used across all domain modules. These types ensure consistent session identity, element representation, and recording event structure throughout the framework.

---

## Functional Requirements

### Element Model

| ID | Requirement |
|----|-------------|
| FR-01-010 | Elements SHALL be represented with at minimum: `elementId`, `role`, `text`, `visible`. |
| FR-01-011 | Element IDs SHALL be scoped to a `pageId` and invalidated on full navigation or explicit page reload. |
| FR-01-012 | Element discovery SHALL prioritize accessibility-oriented attributes (ARIA role, accessible name, visible text) for AI-friendly targeting. |

---

## Non-Functional Requirements

| ID | Requirement |
|----|-------------|
| NFR-01-002 | Type definitions SHALL use flat, descriptive field names suitable for LLM consumption and JSON serialization. |

## Data Models / Types

### BrowserSession

```typescript
interface BrowserSession {
  browserId: string;       // UUID v4
  createdAt: string;       // ISO 8601 timestamp
  headless: boolean;
  recordingEnabled: boolean;
  pageIds: string[];       // child page references
}
```

### PageSession

```typescript
interface PageSession {
  pageId: string;          // UUID v4
  browserId: string;       // parent browser
  url: string;             // current URL (updated on navigation)
  title: string;           // current document title
  createdAt: string;       // ISO 8601 timestamp
}
```

### Element

```typescript
interface Element {
  elementId: string;       // UUID v4, scoped to pageId
  role: string;            // ARIA role or inferred role (button, link, textbox, etc.)
  text: string;            // visible text or accessible name (truncated to 200 chars)
  visible: boolean;        // computed visibility
  tag?: string;            // HTML tag name (optional enrichment)
  selector?: string;       // internal CDP node reference (server-only, not exposed to MCP client)
}
```

### RecordedEvent

```typescript
interface RecordedEvent {
  timestamp: string;       // ISO 8601
  type: 'action' | 'assertion' | 'navigation' | 'screenshot' | 'network' | 'console' | 'error';
  pageId?: string;
  payload: Record<string, unknown>;
}
```

### MCP Success Response Envelope

```typescript
interface McpSuccessResponse<T> {
  ok: true;
  data: T;
}
```

### MCP Error Response Envelope

See [error-model.md](./error-model.md) for the full error envelope and `ErrorCode` union.

```typescript
interface McpErrorResponse {
  ok: false;
  error: {
    code: ErrorCode;
    message: string;       // human-readable, AI-friendly
    details?: Record<string, unknown>;
  };
}
```

### Session ID Conventions

| Field | Format | Generated by |
|-------|--------|--------------|
| `browserId` | UUID v4 | Server on `browser.launch` |
| `pageId` | UUID v4 | Server on `page.create` |
| `elementId` | UUID v4 | Server on element discovery; scoped to `pageId` |

## Acceptance Criteria

| ID | Criterion |
|----|-----------|
| AC-01-001 | Shared TypeScript types compile and are importable by all domain modules. |
| AC-01-006 | `BrowserSession`, `PageSession`, and `Element` interfaces match fields used in modules 02, 03, and 04. |
